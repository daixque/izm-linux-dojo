第8章 レッスン1: 出力リダイレクト

📚 説明
English

コマンドの結果はどこへ行くか

これまで、コマンドを実行すると結果がターミナル画面に表示されていました。第7章の grep や ls も同様です。

この画面への出力を標準出力と呼びます。多くのコマンドは、結果を標準出力に送る仕組みになっています。

結果を画面に出す代わりにファイルに保存したい場面もあります。そのとき使うのがリダイレクト（出力の向きを変える操作）です。

> — ファイルに書き込む（上書き）

$ ls > filelist.txt

ls の結果が画面ではなく filelist.txt に保存されます。ファイルが既にある場合は、中身が上書きされます。元の内容は消えるので注意してください。

>> — ファイルに追記

$ ls documents >> filelist.txt

既存の内容の末尾に追加します。上書きではなく、後ろに足していく形です。

使い分け

記号動き
>ファイルを新しく作る、または中身を上書きする
>>ファイルの末尾に追記する

保存した内容の確認

ファイルに書き込んだ内容は、第3章で学んだ cat で確認できます。

$ cat filelist.txt
