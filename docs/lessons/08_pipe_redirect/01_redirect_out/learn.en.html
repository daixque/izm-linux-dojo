Where Does Command Output Go?

So far, when you ran a command, the result appeared on the terminal screen. Commands like grep and ls from Chapter 7 work the same way.

Output to this screen is called standard output. Most commands are built to send their results to standard output.

Sometimes you want to save the result to a file instead of showing it on screen. That is what redirection does — it changes where output goes.

> — Write to a File (Overwrite)

$ ls > filelist.txt

The output of ls is saved to filelist.txt instead of the screen. If the file already exists, its contents are overwritten. The previous content is lost, so be careful.

>> — Append to a File

$ ls documents >> filelist.txt

New output is added at the end of the existing file rather than replacing it.

When to Use Which

SymbolBehavior
>Create a new file or overwrite existing contents
>>Append to the end of a file

Checking Saved Content

You can view what was written using cat from Chapter 3.

$ cat filelist.txt
