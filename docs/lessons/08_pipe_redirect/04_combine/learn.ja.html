検索・パイプ・リダイレクトを組み合わせる

第7章の検索（grep・find）と、第8章のパイプ・リダイレクトを組み合わせると、実際の作業に近い処理ができます。

たとえば「ログからエラー行だけ取り出してファイルに保存する」「エラーの件数を数えて記録する」といった作業が、1 行のコマンドで書けます。

例: 結果をファイルに保存

$ grep ERROR app.log > errors.txt

grep で ERROR 行だけを取り出し、> で errors.txt に書き出します。画面には何も表示されず、結果がファイルに残ります。

例: パイプ + リダイレクト

$ cat app.log | grep ERROR | wc -l > error_count.txt

左から順に処理が進みます。

  1. cat app.log でログを読み出す
  2. grep ERROR で ERROR 行だけに絞る
  3. wc -l で行数を数える
  4. > error_count.txt で件数をファイルに保存する

作業の流れ

実際の作業では、次のような流れでコマンドを組み合わせることが多いです。

  1. データを読み出す（cat やファイル指定）
  2. 必要な行に絞る（grep）
  3. 加工する（wc など）
  4. 結果を保存する（>）
