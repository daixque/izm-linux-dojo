Connect Commands

A pipe (|) sends one command's output as the next command's input. Data flows through the pipe like water through a tube.

Basic Form

command1 | command2

Example: Count ERROR Lines in a Log

$ cat app.log | grep ERROR | wc -l
2
  1. cat app.log — outputs the file contents
  2. grep ERROR — keeps only ERROR lines
  3. wc -l — counts the lines

No Need for Intermediate Files

With pipes, you can chain processing without creating temporary files in between.
