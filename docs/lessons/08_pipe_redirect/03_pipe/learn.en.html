Connecting Commands with Pipes

📚 Learn
日本語

Connect Commands

A pipe (|) sends one command's output as the next command's input. Data flows through the pipe like water through a tube.

Basic Form

command1 | command2

Example: Filter ERROR Lines from a Log

Start by connecting two commands.

$ cat app.log | grep ERROR
2024-01-01 ERROR: connection failed
2024-01-02 ERROR: timeout
  1. cat app.log — outputs the file contents
  2. grep ERROR — keeps only ERROR lines

The left command's output becomes the right command's input, so only ERROR lines are shown.

Chain Three or More Commands

Pipes are not limited to two commands. You can keep chaining to the right—for example, to count the lines.

$ cat app.log | grep ERROR | wc -l
2

The third command, wc -l (covered in Chapter 3), counts the lines and shows that there are 2 ERROR lines.

No Need for Intermediate Files

With pipes, you can chain processing without creating temporary files in between.
