コマンドの出力を次につなげる

📚 説明
English

コマンド同士をつなげる

1 つのコマンドの結果を、別のコマンドの入力として使いたい場面があります。たとえば「ログファイルから ERROR 行だけ取り出して、件数を数える」といった処理です。

中間結果をいちいちファイルに保存しなくても、コマンド同士をその場でつなげられる仕組みがあります。これがパイプ（|）です。前のコマンドの出力が、次のコマンドの入力へ流れるイメージです。

基本の形

コマンド1 | コマンド2

左のコマンドの標準出力が、右のコマンドの標準入力に渡されます。

例: ログから ERROR を絞り込む

まずは 2 つのコマンドをつなげてみましょう。

$ cat app.log | grep ERROR
2024-01-01 ERROR: connection failed
2024-01-02 ERROR: timeout
  1. cat app.log — ファイルの内容を出力する
  2. grep ERROR — ERROR を含む行だけに絞る

左の出力がそのまま右の入力になるので、ERROR 行だけが表示されます。

3 つ以上もつなげられる

パイプは 2 つだけに限りません。さらに右へつなげて、件数まで数えることもできます。

$ cat app.log | grep ERROR | wc -l
2

3 つ目の wc -l（第3章で学んだ wc）が行数を数え、「ERROR 行が 2 行ある」という結果が表示されます。

パイプの利点

パイプを使えば、一時的なファイルを作らずに処理をつなげられます。データが左から右へ流れていくので、処理の流れも読みやすくなります。
