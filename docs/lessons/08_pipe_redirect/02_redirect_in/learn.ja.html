コマンドへの「入力」とは

前のレッスンでは、コマンドの出力（結果）をファイルに送る > を学びました。今度は逆方向、ファイルの内容をコマンドへの入力として渡す方法を学びます。

コマンドは、キーボードからの入力やファイルの内容を「入力」として受け取り、それを処理します。この入力の入口を標準入力と呼びます。

< — ファイルを入力として渡す

コマンド < ファイル名

< を使うと、指定したファイルの内容がコマンドの標準入力に流れ込みます。キーボードで文字を打ち込む代わりに、ファイルの中身を読み込ませるイメージです。

2 つの渡し方

grep のようにファイルを扱うコマンドには、内容を渡す方法が 2 つあります。

書き方意味
grep ERROR app.loggrep に「このファイルを読んで」とファイル名を渡す
grep ERROR < app.logシェルが app.log の内容を読み、grep の標準入力に流し込む

どちらも grep は app.log の内容を調べます。表示される結果は同じです。違いは「grep 自身がファイルを開く」のか「シェルが内容を渡す」のか、という点です。

入力リダイレクトの例

$ grep ERROR < app.log
2024-01-02 ERROR Connection failed
2024-01-03 ERROR Disk full

この書き方では、grep の後ろにファイル名は書きません。< app.log の部分で入力が渡されます。

いつ使うか

ファイル名を引数に取れるコマンド（grep や cat など）では、通常は grep ERROR app.log のように書けば十分です。

< は、次のレッスンで学ぶパイプ（コマンド同士をつなぐ仕組み）と合わせて理解すると、「データの流れ」が分かりやすくなります。入力リダイレクトは「ファイルからコマンドへデータが流れる」基本形です。
