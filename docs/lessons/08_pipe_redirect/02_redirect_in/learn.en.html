Read Input from a File

While > sends output to a file, < feeds a file's contents as input to a command.

Basic Form

COMMAND < FILENAME

For example, these two mean the same thing:

grep ERROR app.log
grep ERROR < app.log

In the second form, the contents of app.log are passed to grep (in this example, without a filename argument on grep).

When It Is Useful

Before combining with pipes (next lesson), get the idea that < makes a file the input source.
