数字でも権限を書ける

前のレッスンでは chmod u+x のような記号形式で権限を変えました。ほかに、chmod 644 のように数字 3 桁で指定する方法もあります。

Web や書籍で 644 や 755 という数字をよく見かけます。記号形式と同じ権限を、別の書き方で表しているだけです。このレッスンでは、その数字の意味を1 から順に説明します。

まず復習 — ls -l の権限文字列

ls -l の 1 列目は、次の 10 文字で権限を表します（この章の最初のレッスンで学びました）。

-rw-r--r-- 1 user user  ... report.txt
  • 1 文字目 — 種類（-＝ファイル、d＝ディレクトリ）
  • 2〜4 文字目 — オーナーの rwx
  • 5〜7 文字目 — グループの rwx
  • 8〜10 文字目 — その他（others）の rwx

たとえば rw- は「読める・書ける・実行不可」、r-- は「読めるだけ」です。数字形式は、この r w x の ON/OFF を 1 桁の数字にまとめたものです。

3 つの権限を「スイッチ」で考える

1 人分（オーナー、グループ、その他のどれか 1 つ）の権限は、次の 3 つのスイッチで表せます。

スイッチ文字意味
読むr内容を読める
書くw内容を変更できる
実行xプログラムとして実行できる（ディレクトリなら中に入れる）

スイッチが ON なら文字が表示され、OFF なら - になります。

r  ON  /  OFF  →  r  または  -
w  ON  /  OFF  →  w  または  -
x  ON  /  OFF  →  x  または  -

たとえば「読める・書ける・実行不可」は rw- です（r と w が ON、x が OFF）。

2 進数 — 0 と 1 の 3 ビット

コンピュータは ON/OFF を 1 と 0 で表します。3 つのスイッチなら、3 ビット（3 桁の 0 と 1）になります。

左から順に「読む・書く・実行」に対応させます。

位置対応する権限ON のときOFF のとき
左読む r10
中書く w10
右実行 x10

具体例:

  • rw- → r=ON, w=ON, x=OFF → 2 進数 110
  • r-x → r=ON, w=OFF, x=ON → 2 進数 101
  • r-- → r=ON, w=OFF, x=OFF → 2 進数 100
  • --- → 全部 OFF → 2 進数 000

2 進数の「桁」には重みがある

2 進数では、桁の位置によって 1 の価値が変わります。たとえば 10 進数の「一の位・十の位」と同じ考え方です。

2 進数では、右端から左へ 1 桁ずつ、重みが 2 倍になります。3 桁だけ使う場合は次のとおりです。

桁の位置重みこの桁が 1 のとき足される値
右11
中22
左44

左端は 1 でも 1 ではなく 4 です。左に 1 桁増えるたびに重みが 2 倍になる、というルールから 4・2・1 が決まります。権限専用の決まりではありません。

前の表で「左＝r、中＝w、右＝x」と対応づけたので、重みも次のように読めます。

  • 左の桁（r）の重み … 4
  • 中の桁（w）の重み … 2
  • 右の桁（x）の重み … 1

重みを足して 1 桁の数字にする

ON になっているスイッチの重みだけを足します。OFF の桁は 0 なので足しません。

たとえば 110（rw-）なら、左が 1 なので 4、中が 1 なので 2、右が 0 なので 0 → 4+2+0=6 です。

3 ビットで表せる組み合わせは 2×2×2＝8 通り（0〜7）だけです。この 1 桁の数字を8 進数と呼び、chmod ではこの 0〜7 を 1 桁として使います。

2 進数権限文字列重みの足し算1 桁の数字
000---0+0+00
001--x0+0+11
010-w-0+2+02
011-wx0+2+13
100r--4+0+04
101r-x4+0+15
110rw-4+2+06
111rwx4+2+17

文字列と数字の変換

文字列 → 数字（たとえば r-x を 1 桁にする）:

  1. r w x の ON/OFF を確認する
  2. ON の桁の重み（4・2・1）を足す → 4+0+1=5

数字 → 文字列（たとえば 6 を文字列にする）:

  1. 4・2・1 のうち、足してその数字になる組み合わせを探す → 6=4+2
  2. 含まれる重みの桁を ON、それ以外を OFF → rw-

3 桁の数字 — 3 つのグループ

chmod の数字は3 桁です。左から順に、オーナー・グループ・その他について、上で求めた 1 桁ずつを並べます。

  6    4    4
│    │    │
│    │    └── その他: r-- → 4
│    └─────── グループ: r-- → 4
└──────────── オーナー: rw- → 6

644 は、ls -l では次のように見えます。

-rw-r--r--  ... report.txt
 ^^^ ^^^ ^^^
 6   4   4

各桁の計算:

桁対象計算権限文字列
6オーナー4+2rw-
4グループ4r--
4その他4r--

普通のテキストファイルによく使われます。

もう 1 つの例: 755 → オーナー 7=4+2+1（rwx）、グループとその他 5=4+1（r-x）→ rwx r-x r-x。ディレクトリや実行ファイルによく使います。

よく使う数字の一覧

8 進数権限文字列用途の例
644rw- r-- r--普通のテキストファイル
755rwx r-x r-xディレクトリや実行ファイル
600rw- --- ---オーナーだけが読み書きできるファイル
700rwx --- ---オーナーだけが触れるディレクトリ

記号形式との対応: chmod u=rw,go=r file と chmod 644 file は同じ権限です。

使い方

$ chmod 644 report.txt
$ chmod 755 myproject/
$ ls -l report.txt myproject/

実行後は ls -l で 1 列目を確認し、意図した rw- や rwx になっているか確かめましょう。
