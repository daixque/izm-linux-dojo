第4章のおさらい

第4章では「誰のファイルか」（オーナー・グループ）を学びました。第5章では「何が許されているか」＝権限（permissions）を学びます。

3 種類の操作：r・w・x

Linux のファイル権限は、次の 3 つを組み合わせて表します。

  • r（read）— 読む（cat など）
  • w（write）— 書く・変更する（touch、rm など）
  • x（execute）— 実行する。ディレクトリの場合は「中に入る」（cd）

権限がない操作をしようとすると、Permission denied（許可されていません）というエラーになります。

ls -l の権限表示

ls -l の 1 列目が権限です。10 文字で、意味は次のとおりです。

-rw-r--r-- 1 user user   12 Jan  1 10:00 notes.txt
drwxr-xr-x 2 user user 4096 Jan  1 10:00 bin
  • 1文字目 — 種類（-＝ファイル、d＝ディレクトリ）
  • 2〜4文字目 — オーナーの rwx
  • 5〜7文字目 — グループの rwx
  • 8〜10文字目 — その他（others）の rwx

- は「その権限なし」を意味します。たとえば rw- は「読める・書ける・実行不可」です。

具体例

-rw-r--r--  … オーナー: 読・書可 / グループ: 読のみ / その他: 読のみ
-rwxr-xr-x  … オーナー: 読・書・実行可 / グループ: 読・実行可 / その他: 読・実行可
drwx------  … ディレクトリ。オーナーだけが読・書・実行（＝中に入れる）
