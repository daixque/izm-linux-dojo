Review of Chapter 4

Chapter 4 covered whose file something is (owner and group). Chapter 5 covers what is allowed — permissions.

Three Operations: r, w, x

Linux file permissions combine three letters:

  • r (read) — read the file (cat, etc.)
  • w (write) — change the file (touch, rm, etc.)
  • x (execute) — run a file; for directories, enter it (cd)

Without permission, you get Permission denied.

Permission String in ls -l

Column 1 of ls -l is a 10-character permission string:

-rw-r--r-- 1 user user   12 Jan  1 10:00 notes.txt
drwxr-xr-x 2 user user 4096 Jan  1 10:00 bin
  • Char 1 — type (- file, d directory)
  • Chars 2–4 — owner's rwx
  • Chars 5–7 — group's rwx
  • Chars 8–10 — others' rwx

- means that permission is off. rw- means read and write, not execute.

Examples

-rw-r--r--  … owner: read+write / group: read only / others: read only
-rwxr-xr-x  … owner: all / group: read+execute / others: read+execute
drwx------  … directory; only owner can read, write, and enter
