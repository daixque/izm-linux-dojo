記号形式で権限を変更する

📚 説明
English

権限を変更する理由

ファイルを作ると、だいたい次のような初期権限が付きます。

-rw-r--r-- 1 user user  ... memo.txt

オーナーは読み書きできますが、実行はできません。プログラム（シェルスクリプトなど）を動かしたいときは、実行権限（x）を追加する必要があります。

また、秘密のメモを他人に読まれないようにしたいときは、読み取り権限（r）を外す、といった調整もできます。

権限を変更するコマンドが chmod（change mode）です。

誰が変更できるか

自分がオーナーのファイルなら、chmod で権限を変えられます。他人がオーナーのファイルは変えられません（後の章で学ぶ管理者権限が必要な場合があります）。

記号形式 — 3 つのパーツ

記号形式は、「誰に」「何を」「どうする」の 3 つを文字で指定する方法です。

誰に（who）

記号対象
uオーナー（user）
gグループ（group）
oその他（others）
aすべて（all）

どうする（operator）

記号意味
+権限を追加する
-権限を削除する
=権限をその値に設定する（上書き）

何を（what）

r（読む）、w（書く）、x（実行）を組み合わせます。たとえば rx は「読むと実行」の 2 つをまとめて指定します。

例 1 — オーナーに実行権限を追加

スクリプト hello.sh を実行できるようにします。

$ ls -l hello.sh
-rw-r--r-- 1 user user  20 Jan  1 10:00 hello.sh

$ chmod u+x hello.sh

$ ls -l hello.sh
-rwxr-xr-x 1 user user  20 Jan  1 10:00 hello.sh

u+x は「オーナー（u）に実行権限（x）を追加（+）」という意味です。2 文字目が w から x に変わり、rwx になりました。

例 2 — グループとその他から読み取りを削除

$ chmod go-r secret.txt

go-r は「グループ（g）とその他（o）から読み取り（r）を削除（-）」という意味です。秘密のファイルを自分だけが読めるようにできます。

例 3 — すべてに読み取りを追加

$ chmod a+r public.txt

a+r は「すべて（a）に読み取り（r）を追加（+）」という意味です。公開用のファイルによく使います。

例 4 — 権限をまとめて設定

$ chmod u=rw,go=r file

u=rw でオーナーを「読み・書きのみ」に、go=r でグループとその他を「読みのみ」に設定します。= は「今の値に足す」のではなく「この値にそろえる」という意味です。カンマ（,）で複数の指定を続けられます。

変更後は必ず確認

chmod のあとは ls -l で 1 列目の権限文字列を確認する習慣をつけましょう。意図どおり変わっているか、すぐ確かめられます。

注意

自分がオーナーでないファイルの chmod はできません（Operation not permitted）。他人のファイルの権限は、オーナーか管理者だけが変えられます。
