Changing Permissions with chmod

The owner (or root, covered later) can change permissions with chmod. Symbolic mode spells out who, what, and how in letters.

Symbols

  • Who — u owner, g group, o others, a all
  • Operator — + add, - remove, = set exactly
  • What — r, w, x

Common Examples

$ chmod u+x script.sh     # add execute for owner
$ chmod go-r secret.txt   # remove read from group and others
$ chmod a+r public.txt    # add read for everyone
$ chmod u=rw,go=r file    # owner: rw; group and others: r only

Always verify with ls -l after changing permissions.

Note

You cannot chmod a file you do not own (Operation not permitted). Only the owner or an administrator can change permissions.
