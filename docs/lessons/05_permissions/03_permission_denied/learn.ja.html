権限で拒否される

📚 説明
English

Permission denied とは

権限のない操作をしようとすると、Linux はPermission denied（許可されていません）と返します。エラーに見えますが、「セキュリティが正常に働いている」サインでもあります。

よくある場面

  • 他人のファイルを cat しようとした（読み取り権限なし）
  • 他人のファイルを rm しようとした（書き込み権限なし）
  • 権限のないディレクトリに cd しようとした（実行権限なし）
  • 他人のファイルの chmod をしようとした（オーナーでない）
$ cat alice_secret.txt
cat: alice_secret.txt: Permission denied

対処の考え方

  1. ls -l でオーナーと権限を確認する
  2. 自分のファイルなら chmod で権限を調整できる（第5章）
  3. 他人のファイルやシステムファイルなら、オーナーに頼むか、第6章で学ぶ sudo が必要な場合がある

安易に権限を緩めたり、sudo を使ったりしないことが大切です。まずは「なぜ拒否されたか」を理解しましょう。
