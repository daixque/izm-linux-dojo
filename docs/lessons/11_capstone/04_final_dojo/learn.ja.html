English

この演習のゴール

プロジェクト用フォルダを作成し、ログから ERROR を抽出して保存し、README を書く — 一連の作業を自分の力で完了します。第1–9章で学んだファイル操作、検索、パイプ、リダイレクトを組み合わせた、道場必修コースの最終課題です。

課題の流れ

  1. プロジェクト用フォルダ project を作成
  2. ログをコピーし、ERROR 行だけを抽出して保存
  3. 結果ファイルの行数を確認
  4. README を作成して作業完了

使う主なコマンド

ステップコマンドの例
フォルダ作成mkdir -p project/logs
ログをコピーcp server.log project/logs/
ERROR を抽出・保存grep ERROR ... > project/errors.txt
行数を確認wc -l project/errors.txt
README 作成echo "Project complete" > project/README.txt

心がけ

  • 各ステップのあと ls や cat で確認
  • コマンドは一つずつ確実に
  • 困ったら man や --help を参照（第9章）

この演習では rm は使いませんが、今後ファイルを削除するときは、削除前に ls で対象を確認し、削除したファイルは元に戻せないことを忘れないでください。
