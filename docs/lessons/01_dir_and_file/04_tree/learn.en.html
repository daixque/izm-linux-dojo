Tree Structure — Folders Inside Folders

The Linux file system is organized as a branching hierarchy, like a tree. The root (/) is the trunk; directories branch out, and files sit at the ends of those branches.

Mac Finder's sidebar and the file explorer on the exercise page show the same hierarchy in a different view. Being able to read a tree diagram makes it easier to tell "which level am I on?" in any display.

Here is a deeper version of the example from Lesson 1:

/home/user/
├── projects/
│   └── web/
│       └── app/
│           └── config.txt
└── notes.txt

Inside projects is web, inside that is app, and inside that is config.txt — a structure three folders deep.

Counting Levels

"Three levels down" means passing through three directories from where you start.

From /home/user to config.txt:

  1. projects — level 1
  2. web — level 2
  3. app — level 3 (where config.txt lives)

You cannot cd into a file — only into directories. The goal is to reach the directory that contains config.txt (that is, app).

Reaching Deep Directories

You can move in one step or one level at a time.

One combined path (recommended)

$ cd projects/web/app
$ pwd
/home/user/projects/web/app

Join directory names with / to jump to a deep location at once. This works with both relative and absolute paths.

Step by step

$ cd projects
$ cd web
$ cd app
$ pwd
/home/user/projects/web/app

Use this when you want to verify each level. Either way, finish with pwd to confirm where you are.

Reading Tree Diagrams

Documentation often shows structure with tree diagrams like the one above.

  • ├── — a branch with siblings at the same level
  • └── — the last item at that level
  • │ — the branch continues from above

Keeping track of "which level am I on?" while reading a tree makes cd and ls output easier to follow.

Tab Completion Shines Here

Deep paths get long. Type cd pro and press Tab to get projects/, then w + Tab for web/, and so on. You do not have to type the full path by hand — get comfortable with Tab.
