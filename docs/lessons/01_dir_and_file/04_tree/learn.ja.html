フォルダの階層構造を把握する

📚 説明
English

ツリー構造 — フォルダの入れ子

Linux のファイルシステムは、木（ツリー）のように枝分かれした階層になっています。ルート（/）が木の根元で、そこからディレクトリが枝のように伸び、先端にファイルがぶら下がるイメージです。

Mac の Finder サイドバーや、演習ページ右側のファイルエクスプローラーも、同じ階層を別の見え方で示しています。

第1レッスンで見た例を、もう少し深くしてみましょう。

/home/user/
├── projects/
│   └── web/
│       └── app/
│           └── config.txt
└── notes.txt

projects の中に web、その中に app、さらにその中に config.txt という3 階層の入れ子になっています。

階層の数え方

「3 階層下」とは、今いる場所から数えて 3 つディレクトリを通過することを指します。

/home/user から config.txt までの道のり：

  1. projects — 1 階層目
  2. web — 2 階層目
  3. app — 3 階層目（ここに config.txt がある）

ファイルそのものには cd できません（ディレクトリだけが移動先になります）。config.txt のあるディレクトリ（app）まで移動する、というのが正しい操作です。

深い階層への移動方法

一気に移動する方法と、一つずつ移動する方法があります。

パスをまとめて指定（おすすめ）

$ cd projects/web/app
$ pwd
/home/user/projects/web/app

スラッシュ / でディレクトリ名をつなげれば、深い場所へ一度に移動できます。相対パスでも絶対パスでも同じ考え方です。

一つずつ移動

$ cd projects
$ cd web
$ cd app
$ pwd
/home/user/projects/web/app

階層を一つずつ確認しながら進めたいときに便利です。どちらの方法でも、最後に pwd で現在地を確認しましょう。

tree コマンド — 構造を一覧表示

ターミナルからディレクトリ構造をツリー形式で表示したいときは、tree コマンドを使います。多くの Linux 環境には最初から入っています。macOS では Homebrew などで別途インストールする場合があります。

$ tree
.
├── projects/
│   └── web/
│       └── app/
│           └── config.txt
└── notes.txt

引数なしで実行すると、現在のディレクトリ以下をまとめて表示します。特定のディレクトリを指定することもできます。

$ tree projects

ls はその場の一覧、tree は全体像——GUI のファイルエクスプローラーに近い見え方です。

Tab 補完が特に便利

深い階層ではパスが長くなります。cd pro と入力して Tab を押すと projects/ まで補完され、続けて w + Tab で web/ … と進められます。長いパスを全部手打ちしなくて済むので、Tab に慣れておきましょう。
