ls と cd の基本

📚 説明
English

ディレクトリとファイル

Linux 上のデータは、ディレクトリ（フォルダ）とファイルの 2 種類に分かれています。

  • ディレクトリ — ファイルや他のディレクトリを入れておく入れ物
  • ファイル — テキストや画像など、実際のデータそのもの

ディレクトリの中にさらにディレクトリやファイルを置けるので、入れ子の階層構造になります。たとえば次のような構造です。

/
└── home/
    └── user/
        ├── documents/
        │   └── readme.txt
        └── notes.txt

ファイルやディレクトリの場所を表す文字列をパス（path）と呼びます。パスでは /（スラッシュ）でディレクトリの階層が区切られます。先頭の / はルート（最上位）のディレクトリを表します。

たとえば /home/user は、ルートの中の home ディレクトリ、その中の user ディレクトリを指します（2 つのディレクトリ名が / でつながっています）。readme.txt のパス /home/user/documents/readme.txt も同様に、/ ごとに通過するディレクトリが示されていきます。

コマンドで確認する

このような構造を実際の Linux 上で確認するには、コマンドを使います。

  • 今どのディレクトリにいるかを調べる
  • ディレクトリの中に何があるかを一覧表示する
  • 別のディレクトリへ移動する

これから学ぶ pwd・ls・cd は、まさにこの確認のためのコマンドです。

ターミナルとは？

コマンドを実行するには、ターミナル（端末）という画面が必要です。ターミナルは、キーボードからコマンドを入力し、結果を表示する入出力の場所です。

コマンドを入力したら Enter キーを押すと実行されます。以降の例は、すべてターミナル上での表示です。

基本コマンド

pwd — 現在地を表示

今いるディレクトリのパス（住所）を表示します。

$ pwd
/home/user

ls — ファイル一覧を表示

カレントディレクトリの中身を一覧表示します。user ディレクトリ（パス /home/user）の中身は、次のように表示されます。

$ ls
documents
notes.txt

cd — ディレクトリ移動

別のディレクトリに移動します。

  • cd documents — documents フォルダに移動（相対パス）
  • cd .. — 一つ上のディレクトリに移動
  • cd /home/user — 絶対パスで移動
