Directories and Files

On Linux, data is stored as either directories (folders) or files.

  • Directory — a container that holds files and other directories
  • File — the actual data, such as text or images

Directories can contain more directories and files, forming a nested hierarchy. Here is an example:

/
└── home/
    └── user/
        ├── documents/
        │   └── readme.txt
        └── notes.txt

The string that describes where a file or directory lives is called a path. In a path, each directory level is separated by / (a forward slash). The leading / represents the root (top-level) directory.

For example, /home/user means the home directory inside root, then the user directory inside home — two directory names joined by /. The path to readme.txt, /home/user/documents/readme.txt, works the same way: each / marks another directory you pass through.

Checking with Commands

To verify this structure on a real Linux system, you use commands.

  • Find out which directory you are in
  • List what is inside a directory
  • Move to another directory

The commands pwd, ls, and cd that you will learn next are exactly for this kind of checking.

What is a Terminal?

To run commands, you need a screen called a terminal. A terminal is where you type commands with the keyboard and read the results.

Press Enter after typing a command to run it. The examples below all show what you would see in a terminal.

Basic Commands

pwd — Print Working Directory

Shows the path of your current directory.

$ pwd
/home/user

ls — List Files

Lists the contents of the current directory. Inside the user directory (path /home/user), you would see:

$ ls
documents
notes.txt

cd — Change Directory

Move to another directory.

  • cd documents — move into documents (relative path)
  • cd .. — move up one directory
  • cd /home/user — move using an absolute path
