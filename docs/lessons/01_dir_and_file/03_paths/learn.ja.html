パスの理解

📚 説明
English

パスとは？

パス（path）は、ファイルやディレクトリの「住所」を表す文字列です。第1レッスンで /home/user/documents/readme.txt のような例を見ました。

Finder でフォルダを開いたとき、ウィンドウ下やタイトルバー（⌘+クリック）に表示される場所の文字列も、パスです。ターミナルでは同じ住所を / で区切った形で書きます。

パスの書き方には、大きく 2 種類あります。

絶対パス — ルートからの完全な住所

絶対パスは、ファイルシステムの最上位である ルート（/）から、目的の場所までをすべて書いたパスです。たとえば、郵便物に「都道府県から番地まで全部書く」イメージです。

/home/user/documents/readme.txt

先頭の / が「ルートから始まる」印です。どこにいても、同じ絶対パスは常に同じ場所を指します。

/
└── home/
    └── user/
        ├── documents/
        │   └── readme.txt   ← /home/user/documents/readme.txt
        └── notes.txt        ← /home/user/notes.txt

相対パス — 今いる場所からの道順

相対パスは、今いるディレクトリ（いま開いているフォルダ）を起点にしたパスです。先頭に / は付きません。「今ここから、右に 2 つ目の角を曲がる」ような書き方です。

たとえば /home/user にいるとき：

  • documents — 同じ階層の documents フォルダへ（/home/user/documents と同じ場所）
  • documents/readme.txt — documents の中の readme.txt へ

今いる場所が変わると、同じ相対パスが指す場所も変わります。だから pwd で現在地を確認する習慣が大切です。

~ — ホームディレクトリの省略記号

~（チルダ）は、あなたのホームディレクトリを表す省略記号です。演習では /home/user がホームです。

  • ~ → /home/user
  • ~/documents → /home/user/documents

プロンプトに ~ と表示されているとき、ホームにいる合図です。Mac でも同じ考え方で、ホームは /Users/あなたの名前 になります。

. と .. — 特殊なディレクトリ名

すべてのディレクトリには、2 つの特殊な名前が決まっています。

  • .（ドット）— 今いるディレクトリ自身。cd . は移動しない（同じ場所に留まる）ことと同じです。
  • ..（ドットドット）— 一つ上のディレクトリ（親ディレクトリ）。Finder で「戻る」に近い操作が cd .. です。

これらは組み合わせて使えます。

  • cd ../documents — 一つ上に上がってから documents へ
  • cd ./documents — 今の場所から documents へ（cd documents と同じ意味）

絶対パスと相対パスの使い分け

どちらも cd に渡せます。

絶対パスの例：

$ cd /home/user/documents

相対パスの例（/home/user にいる前提）：

$ cd documents

~ を使った例：

$ cd ~/documents

迷ったら pwd で現在地を確認してから cd しましょう。

Tab 補完とコマンド履歴

長いパスを打つときは Tab キーでディレクトリ名を補完できます。以前使った cd コマンドは ↑ キーで呼び出せます。第9章でも扱いますが、パスの練習中にぜひ試してみてください。
