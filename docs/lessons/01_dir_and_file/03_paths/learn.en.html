What Is a Path?

A path is a string that describes the "address" of a file or directory. In Lesson 1 you saw examples like /home/user/documents/readme.txt.

When you open a folder in Finder, the location shown in the path bar or title bar (⌘-click) is also a path. In the terminal, the same address is written with slashes / separating each level.

There are two main ways to write paths.

Absolute Path — Full Address from Root

An absolute path starts at the top of the file system — the root (/) — and lists every directory down to the destination. Think of it like writing a full mailing address from the country down to the street number.

/home/user/documents/readme.txt

The leading / means "starting from root." No matter where you are, the same absolute path always points to the same place.

/
└── home/
    └── user/
        ├── documents/
        │   └── readme.txt   ← /home/user/documents/readme.txt
        └── notes.txt        ← /home/user/notes.txt

Relative Path — Directions from Where You Are

A relative path starts from your current directory (the folder you are in right now). It does not begin with /. It is like saying "from here, turn right at the second corner."

When you are in /home/user:

  • documents — enter the documents folder here (same as /home/user/documents)
  • documents/readme.txt — reach readme.txt inside documents

Because the starting point changes, the same relative path can lead to different absolute locations. That is why checking with pwd is a good habit.

~ — Shorthand for Home

~ (tilde) stands for your home directory. In the exercises, home is /home/user.

  • ~ → /home/user
  • ~/documents → /home/user/documents

When the prompt shows ~, you are in your home directory. On Mac, home is /Users/yourname, but the idea is the same.

. and .. — Special Directory Names

Every directory has two special names:

  • . (dot) — the current directory itself. cd . means stay where you are.
  • .. (dot-dot) — the parent directory (one level up). In Finder, this is close to going "back." cd .. moves up one level.

You can combine them:

  • cd ../documents — go up one level, then into documents
  • cd ./documents — from here into documents (same as cd documents)

Choosing Absolute vs Relative

Both work with cd.

Absolute path example:

$ cd /home/user/documents

Relative path example (while in /home/user):

$ cd documents

Example using ~:

$ cd ~/documents

When unsure, run pwd first, then cd.

Tab Completion and Command History

For long paths, press Tab to complete directory names. Previously used cd commands can be recalled with the ↑ key. Chapter 9 goes deeper, but try these while practicing paths.
