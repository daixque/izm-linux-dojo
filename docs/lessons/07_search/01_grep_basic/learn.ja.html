ファイルの中から行を探す

第3章で cat や head・tail を使い、ファイルの中身を読む方法を学びました。ログや設定ファイルは行数が多いことがあり、必要な行だけを素早く見つけたい場面があります。

たとえばエラーログでは、ERROR という文字列を含む行だけを取り出したい、ということがよくあります。1 行ずつ目で探すのは大変なので、文字列を含む行だけを自動で表示する仕組みがあります。

grep とは

grep は、ファイルの中から指定した文字列を含む行だけを取り出して表示するコマンドです。テキストファイルの検索に使います。

基本の形

grep パターン ファイル名

「パターン」には探したい文字列を書きます。大文字と小文字は区別されます（ERROR と error は別物として扱われます）。

$ grep ERROR app.log
2024-01-02 ERROR Connection failed
2024-01-03 ERROR Disk full

上の例では、app.log の各行を調べ、ERROR という文字列を含む行だけが表示されています。

複数ファイルをまとめて検索

ファイル名を複数指定すると、それぞれのファイルを順番に検索します。

$ grep ERROR log1.txt log2.txt

見つからないとき

パターンに一致する行がない場合、grep は何も表示せず終了します。これはエラーではなく、「該当する行がなかった」という結果です。
