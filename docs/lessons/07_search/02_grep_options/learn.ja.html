検索の条件を細かく変える

前のレッスンでは grep パターン ファイル名 の基本形を学びました。コマンドの後ろにオプション（追加の指定）を付けると、検索の仕方を調整できます。

オプションはハイフン（-）から始まる短い文字列です。よく使うものを次に示します。

主なオプション

オプション意味
-i大文字と小文字を区別しない
-n見つかった行の行番号も表示する
-vパターンに一致しない行を表示する
-c一致した行の件数だけを表示する

-i — 大文字小文字を無視

$ grep -i error app.log

ERROR も error も Error も一致します。

-n — 行番号を表示

$ grep -n ERROR app.log
4:2024-01-02 ERROR Connection failed
6:2024-01-03 ERROR Disk full

何行目に見つかったかが分かります。あとで該当箇所を開くときに便利です。

-v — 一致しない行を表示

$ grep -v ERROR app.log

ERROR を含まない行だけが表示されます。エラー以外の行を見たいときに使います。

-c — 件数だけを表示

$ grep -c ERROR app.log
2

行の内容ではなく、何件見つかったかだけを知りたいときに便利です。

オプションの書き方

オプションはパターンの前に書きます。

grep -i -n パターン ファイル名

複数のオプションをまとめて grep -in パターン ファイル名 のように書くこともできます。
