Filename Patterns

The * in find -name "*.txt" is called a wildcard. It lets you specify part of a filename loosely. Before the command runs, wildcards are replaced with actual filenames. This replacement is called expansion.

Common Wildcards

SymbolMeaningExample
*Any string (0 or more characters)*.txt → notes.txt, readme.txt
?Any single characterfile?.txt → file1.txt, fileA.txt

Example: ls *.txt

$ ls *.txt
notes.txt  readme.txt

Before the command runs, *.txt is replaced with filenames ending in .txt in the current directory, and then ls runs. It is the same as running ls notes.txt readme.txt.

Wildcards vs find

  • Wildcards — expanded automatically before the command runs (usually one directory level, in the current folder)
  • find — searches through subdirectories too
