移動と名前変更 — OS の見方

コンピューターから見ると、ファイルの移動も名前変更も、「そのファイルの住所（パス）を変える」操作です。同じフォルダの中で名前だけ変えるのは、住所の末尾部分だけ書き換えるのと同じ考え方です。

Finder では、ファイルを別フォルダへドラッグするのが移動、ファイル名をクリックして編集するのがリネームです。ターミナルでは mv（move）コマンド 1 つで両方できます。

cp との違い

  • cp — 元を残してコピーを作る（複写）
  • mv — 元の場所からなくなり、新しい場所（または新しい名前）に移る

名前を変える — リネーム

$ mv old_name.txt new_name.txt
$ ls
new_name.txt

同じディレクトリ内で、ファイル名だけが old_name.txt から new_name.txt に変わります。

別のフォルダへ移動する

$ mv new_name.txt archive/
$ ls archive
new_name.txt

ファイルが archive フォルダの中に移動します。元の場所からは消えます。

移動と名前変更を同時に

移動先で別名を付けることもできます。

$ mv draft.txt archive/final.txt

これで draft.txt は archive/final.txt という名前で移動します。
