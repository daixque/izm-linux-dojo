Moving and Renaming — How the OS Sees It

From the computer's point of view, both moving and renaming a file mean changing its address (path). Renaming within the same folder is like rewriting only the last part of the address.

In Finder, dragging a file to another folder is a move; clicking the filename to edit it is a rename. In the terminal, the single mv (move) command handles both.

Difference from cp

  • cp — keeps the original and creates a copy
  • mv — the item leaves its old location and appears at the new one (or under a new name)

Rename a File

$ mv old_name.txt new_name.txt
$ ls
new_name.txt

Within the same directory, the filename changes from old_name.txt to new_name.txt.

Move to Another Folder

$ mv new_name.txt archive/
$ ls archive
new_name.txt

The file moves into archive and disappears from its original location.

Move and Rename at Once

You can also give the file a new name at the destination.

$ mv draft.txt archive/final.txt

draft.txt becomes archive/final.txt in one step.
