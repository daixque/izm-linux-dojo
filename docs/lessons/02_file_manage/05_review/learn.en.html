What You Have Learned (Chapters 1–2)

You now have the core file operations for the terminal. Here is a recap.

Chapter 1 — Navigation

CommandMeaning
pwdShow current directory path
lsList directory contents
ls -aList including hidden files
cdChange directory

Paths can be absolute (from root /) or relative (from where you are). ~ means home; .. means one level up.

Chapter 2 — File Management

CommandMeaning
mkdirCreate a directory
mkdir -pCreate a directory hierarchy (parents included)
touchCreate an empty file
cpCopy a file (original remains)
cp -rCopy a directory
mvMove or rename (original gone)
rmDelete a file
rm -r / rmdirDelete a directory

Typical Workflow

In real work, you combine commands in a flow like this:

  1. Use pwd and ls to see where you are
  2. Use mkdir -p to prepare folders
  3. Use cp and mv to organize files
  4. Use rm to clean up
  5. Run ls again to verify the result

Common Mistakes

  • Wrong directory — always check with pwd first
  • Confusing cp and mv — copy keeps the original, move does not
  • Deleting too quickly — read filenames and paths carefully
