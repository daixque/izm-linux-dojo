Deleting Files and Directories

📚 Learn
日本語

What Is Deletion?

Deletion removes files or directories you no longer need. On Linux you use rm (remove).

In Finder, moving a file to the Trash temporarily holds it so you can restore it until you empty the Trash. Terminal rm usually deletes directly without going through a trash bin. Deleted data is hard to recover, so always double-check before pressing Enter.

Delete a File — rm

$ rm temp.txt

The specified file is removed. Use ls afterward to confirm it is gone.

Delete a Directory — rm -r

When deleting a folder, you need the -r option. It removes everything inside, including subfolders.

$ rm -r old_folder

For an empty folder only, you can use rmdir instead (it fails if the folder has contents).

$ rmdir empty_dir

Safety Tips

  • Confirm it is safe to delete — read the filename carefully before pressing Enter
  • Do not mistype the path — run pwd to confirm where you are before deleting
  • Be extra careful with directories — rm -r removes all contents

On real Linux systems, dangerous commands like rm -rf / have destroyed entire systems. Build the habit of verifying paths first.
