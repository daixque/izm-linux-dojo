Copying Files and Directories

📚 Learn
日本語

What Is Copying?

Copying a file or folder means creating a duplicate while keeping the original. Think of photocopying a book — you end up with both the original and the copy.

On Mac Finder, selecting a file and pressing ⌘+C (Copy) then ⌘+V (Paste) in another location does the same thing. On Linux you use the cp (copy) command.

Copy a File — cp

The basic form is:

cp source destination

To copy report.txt into the backup folder:

$ cp report.txt backup/
$ ls backup
report.txt

The original report.txt stays in place, and a new file appears at backup/report.txt.

Copy with a New Name

If you specify a filename as the destination, you can create a copy under a different name.

$ cp report.txt report_backup.txt

Copy a Directory — cp -r

When copying a folder, you need the -r option. It copies everything inside, including subfolders.

$ cp -r photos photos_copy
$ ls
photos
photos_copy

Without -r, you usually get an error saying directories cannot be copied.

Verify the Copy

Use ls from Chapter 1 or cat from Chapter 3 to confirm the copy exists. You can also cd into the destination folder first.
