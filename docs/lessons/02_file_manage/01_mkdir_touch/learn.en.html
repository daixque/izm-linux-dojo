Creating Files and Directories

Review from Chapter 1

In Chapter 1 you learned pwd, ls, and cd — commands to see where you are, list contents, and move around.

This chapter covers how to create new files and directories.

Creating a Directory — mkdir

mkdir (make directory) creates a new folder.

$ mkdir project
$ ls
documents
notes.txt
project

A folder named project appears inside your current directory.

Creating nested paths — mkdir -p

Without -p, mkdir project/src fails if project does not exist yet. The -p option creates any missing parent directories along the way.

$ mkdir -p project/src
$ ls project
src

Use this when you want a deep path like photos/2024/summer in one step.

Creating an Empty File — touch

touch creates a new empty file. Think of it as preparing a blank notebook before you write in it.

$ touch memo.txt
$ ls
documents
memo.txt
notes.txt
project

If the file already exists, touch updates its timestamp without changing the contents.

Verify with ls

Use ls from Chapter 1 to confirm your new items exist. If something looks wrong, run pwd — beginners often create files in the wrong directory by accident.
