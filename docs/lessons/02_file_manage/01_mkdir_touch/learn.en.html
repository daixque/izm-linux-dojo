Creating Files and Directories

Review from Chapter 1

In Chapter 1 you learned pwd, ls, and cd — how to see where you are, list contents, and move around. These are "look and move" operations.

This chapter covers how to create new files and directories.

Creating New Folders and Files

In Finder, "File → New Folder" or "New File" adds an item in the folder you currently have open. The terminal can do the same thing — new items appear in your current directory (the location shown by pwd).

Before you create something, run pwd to confirm where you are. That helps prevent accidentally creating files in the wrong place.

Creating a Directory — mkdir

mkdir (make directory) creates a new folder.

$ mkdir project
$ ls
documents
notes.txt
project

A folder named project appears inside your current directory.

When the parent folder does not exist

If you try something like mkdir project/src when project does not exist yet, you may get an error. You need to create project first, then src.

Creating deep paths at once — mkdir -p

The -p option creates any missing parent directories along the way.

$ mkdir -p project/src
$ ls project
src

Use this when you want a deep path like photos/2024/summer without creating each folder one by one.

Creating an Empty File — touch

touch creates a new empty file — like preparing a blank notebook before you write in it. You can add content later with a text editor.

$ touch memo.txt
$ ls
documents
memo.txt
notes.txt
project

Verify with ls

Use ls from Chapter 1 to confirm your new items exist. If something looks wrong, run pwd to check which directory you are in.
