ファイルのオーナーを変更する

📚 説明
English

オーナーを変える必要がある場面

ファイルのオーナーは、だいたい「作った人」になります。でも、次のような場面ではオーナーを変える必要があります。

  • 管理者（root）が作ったファイルを、利用者に渡したい
  • 退職した人のファイルを、別の担当者に引き継ぎたい
  • 共有フォルダのファイルを、チームのメンバーが管理できるようにしたい

オーナーを変更するコマンドが chown（change owner）です。グループだけを変える chgrp（change group）もあります。

chown — オーナーを変更

$ ls -l admin.log
-rw-r--r-- 1 root root  50 Jan  1 10:00 admin.log

$ sudo chown user admin.log

$ ls -l admin.log
-rw-r--r-- 1 user root  50 Jan  1 10:00 admin.log

オーナーが root から user に変わりました。これで user が自分のファイルとして扱えるようになります（権限の設定次第で読み書きも可能になります）。

オーナーとグループを同時に変更

$ sudo chown user:staff admin.log

ユーザー名:グループ名 の形式で、オーナーとグループを一度に指定できます。上の例では、オーナーを user、グループを staff に変更します。

chgrp — グループだけ変更

$ sudo chgrp staff admin.log

オーナーはそのままで、グループだけ staff に変えたいときに使います。

なぜ sudo が必要か

他人がオーナーのファイルの chown / chgrp は、普通のユーザーにはできません。root だけが任意のオーナーに変更できます。

$ chown user admin.log
chown: changing ownership of 'admin.log': Operation not permitted

$ sudo chown user admin.log

1 行目は sudo なしで失敗し、2 行目は sudo を付けることで成功します。

自分がオーナーのファイルなら、グループの変更（chgrp）は sudo なしでできる場合もありますが、root オーナーのファイルには sudo が必要です。

典型例 — 管理者が作ったファイルを渡す

管理者が admin.log を作り、利用者 user に管理を任せたい場面を想像してください。

  1. sudo chown user admin.log でオーナーを user に変更
  2. ls -l admin.log で 3 列目が user になったことを確認
  3. 必要なら chmod で権限も調整
