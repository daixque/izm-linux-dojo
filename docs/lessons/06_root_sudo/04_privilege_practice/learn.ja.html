特権操作の実践

English

ユーザー・権限・sudo を組み合わせる

ユーザー・オーナーシップ・権限・root・sudo は、実際の現場では組み合わせて使います。共有フォルダを整備する場面を例に、流れを確認しましょう。

共有フォルダの整備シナリオ

管理者が /shared という共有フォルダを整備する場面を想像してください。

  1. フォルダのオーナーを利用者 user に変更（sudo chown）
  2. メンバー全員が読めるように権限を設定（sudo chmod）
  3. ls -l で結果を確認
$ sudo chown user:staff /shared/data.txt
$ sudo chmod 664 /shared/data.txt
$ ls -l /shared/data.txt
-rw-rw-r-- 1 user staff  ... data.txt

オーナーが user、グループが staff、権限が 664（オーナーとグループは読書可、その他は読みのみ）になっています。

大切な心構え

  • sudo は必要なときだけ — まず普通の方法でできるか考える
  • 変更前後で ls -l を確認する
  • 権限は必要最小限に — 広げすぎない
