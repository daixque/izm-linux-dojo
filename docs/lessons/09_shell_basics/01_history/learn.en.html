Terminal and Shell

The terminal you've used since Chapter 0 is the text-based screen itself. It is the window for input and output—it receives what you type and shows the results.

Behind the terminal runs a program that reads what you type and passes commands to the computer. That program is called the shell.

What the Shell Does

When you press Enter, the shell follows these steps:

  1. Read the text you typed (the command)
  2. Replace wildcards with actual file names
  3. Find and run the matching program (such as ls or grep)
  4. Send the output back to the terminal screen

For example, when you type ls *.txt, the shell first expands wildcards such as * and ? in *.txt to something like notes.txt readme.txt, then runs ls (the expansion you learned in Chapter 7). Wildcard expansion is the shell's job, not the command's.

When you ran ls or cd since Chapter 0, the shell handled everything from reading your input to showing the result. Remember: the terminal is the screen; the shell is the program that processes commands.

The shell does more than run commands—it also offers features that help you type. Here we start with command history, which lets you recall commands you typed before.

Recall Commands You Typed

When you started using the terminal in Chapter 1, you may have noticed that the ↑ key brings back previous commands. Here we cover that feature in more detail.

What Is History?

The shell records commands you run after pressing Enter as history. You do not need to retype long commands every time.

↑ and ↓ Keys

  • ↑ — show the previous command (press again for older ones)
  • ↓ — move toward newer commands (use after going back with ↑)

Tips

  • When running the same command repeatedly, press ↑ and Enter
  • When you need a small change, press ↑, edit, then Enter
