Goal of This Exercise

Create a project folder, extract ERROR lines from a log and save them, and write a README — completing the full workflow on your own. This is the final required task of the dojo, combining file operations, search, pipes, and redirection from Chapters 1–9.

Task Flow

  1. Create a project folder project
  2. Copy the log and extract ERROR lines to a file
  3. Check the line count of the result file
  4. Create a README to mark completion

Main Commands

StepExample command
Create foldermkdir -p project/logs
Copy logcp server.log project/logs/
Extract and save ERRORgrep ERROR ... > project/errors.txt
Check line countwc -l project/errors.txt
Create READMEecho "Project complete" > project/README.txt

Mindset

  • After each step, verify with ls or cat
  • Run commands one at a time, carefully
  • When stuck, check man or --help (Chapter 9)

This exercise does not use rm, but when you delete files in the future, confirm the target with ls first — deleted files cannot be easily restored.
