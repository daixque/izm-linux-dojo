ログ分析

📚 説明
English

この演習のゴール

ログファイルから ERROR の行だけを取り出し、件数を数えます。第7–8章で学んだ grep、wc、パイプ（|）を組み合わせて、実務でよくある「エラーが何件あったか調べる」作業を体験します。

ログファイルとは

プログラムやサーバーは、動作の記録をログ（log）ファイルに書き込みます。問題が起きたとき、いつ・何が起きたかを後から調べる材料になります。

ログの各行には、重要度を示すラベルが付いていることが多いです。

ラベル意味
INFO通常の動作記録
WARN警告（問題の可能性）
ERRORエラー（処理が失敗した）

よく使う組み合わせ

$ grep ERROR app.log
$ grep ERROR app.log | wc -l

1 行目は ERROR 行だけを表示、2 行目は ERROR の件数を数えます。パイプ（|）は、前のコマンドの出力を次のコマンドに渡す記号です。

作業の流れ

  1. cat や head でログの全体像を把握
  2. grep で ERROR 行を抽出
  3. パイプで wc -l に渡して件数を数える

次のステップ

演習では app.log から ERROR の件数を集計します。
