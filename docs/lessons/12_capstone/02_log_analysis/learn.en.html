Goal of This Exercise

Extract only ERROR lines from a log file and count them. You will combine grep, wc, and pipes (|) from Chapters 7–8 in a task common in real work: "how many errors were there?"

What Is a Log File?

Programs and servers write activity records to log files. When something goes wrong, logs help you investigate what happened and when.

Each line in a log often has a label showing severity:

LabelMeaning
INFONormal operation record
WARNWarning (possible problem)
ERRORError (processing failed)

Common Combinations

$ grep ERROR app.log
$ grep ERROR app.log | wc -l

The first shows only ERROR lines; the second counts how many ERROR lines exist. The pipe (|) passes the output of the previous command to the next one.

Workflow

  1. Use cat or head to understand the log
  2. grep to extract ERROR lines
  3. Pipe to wc -l to count them
