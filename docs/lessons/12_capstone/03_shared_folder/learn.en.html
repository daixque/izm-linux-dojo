Capstone: Shared Folder

Use users, permissions, and sudo from Chapters 4–6 to set up a shared directory.

Scenario

Prepare /shared so team member user can read and write. Initially it is owned by root and not writable by regular users.

Commands Used

  • sudo chown user:user /shared — change owner and group to user
  • chmod u+rwx,go+rx /shared — user read/write/execute, others read/execute
  • ls -l /shared — verify changes

Why sudo?

Changing ownership of root-owned paths requires administrator privileges — use sudo from Chapter 6.
