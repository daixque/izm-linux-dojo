共有フォルダの権限を整備する

📚 説明
English

この演習のゴール

/shared ディレクトリのオーナーと権限を変更し、チームメンバー user が読み書きできる状態にします。第4–6章で学んだ chown、chmod、sudo を組み合わせて、複数人で使うフォルダを整備する体験をします。

シナリオ

/shared ディレクトリを、チームメンバー user が読み書きできる状態にします。初期状態では root がオーナーで、一般ユーザーは書き込めません。

演習の最後では、user として team.txt を作成できることを確認します。

使うコマンド

コマンド意味
sudo chown user:user /sharedオーナーとグループを user に変更
chmod u+rwx,go+rx /shareduser は読み書き実行、他は読み取りと実行
ls -l /shared変更結果を確認

なぜ sudo が必要？

他人（ここでは root）のファイルのオーナーを変える操作は、管理者権限が必要です。第6章で学んだ sudo を使います。

権限の変更は、他のユーザーのアクセスに影響するため、本番環境では変更前に ls -l で現状を確認し、変更後も結果を確かめましょう。
