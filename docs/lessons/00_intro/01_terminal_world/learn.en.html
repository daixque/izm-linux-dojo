Welcome to IZM Linux Dojo

In this dojo, you will learn to give your computer instructions (commands) through a screen called the terminal. At first you might wonder, "Why use this old-fashioned way?" But once you understand commands, you can organize files, check settings, and handle many tasks precisely and quickly.

In Chapter 0, we will not jump into difficult operations. First, let's build a mental picture of what a terminal is and how Linux and macOS relate to it.

What Are Linux and macOS?

Linux is one of the operating systems (OS) used around the world. Many computers around you — servers, smartphones (Android), embedded devices — run a Linux-based OS.

macOS is the OS used on Apple's Mac computers. It looks different from Linux on the surface, but underneath it is a Unix-like system, very similar to Linux. That means basic terminal operations and the idea of files work almost the same on Linux and macOS.

This dojo mainly uses Linux as an example, but Mac users can follow along with the same intuition.

Note: Windows uses a different path and permission model, so it is outside the scope of this dojo.

GUI and CUI — Two Ways to Operate a Computer

There are two broad ways to operate a computer:

  • GUI (Graphical User Interface) — the familiar screen with icons, buttons, and mouse control. Opening a folder in Finder (Mac) or File Explorer (Windows) is GUI.
  • CUI (Character User Interface) — a text-only screen where you type instructions from the keyboard. The terminal is the classic example of CUI.

GUI is intuitive, but repeating the same work many times or specifying fine-grained conditions can become tedious. CUI (the terminal) takes getting used to, but its strength is precise control with short commands.

Reading the Terminal Screen

On the exercise page, a terminal appears on the left (or center). A typical terminal looks like this:

Welcome to IZM Linux Dojo
Type commands and press Enter.

user@dojo:~$ _
  • Welcome message — a greeting from the dojo. You do not need to press Enter after reading it.
  • Prompt — a marker like user@dojo:~$ that shows the terminal is waiting for your input. You type commands to the right of it.
  • Cursor — the blinking _ shows where the next character will appear.

The ~ in the prompt means you are in your home directory (your personal folder). You will learn more in Chapter 1.

The Enter Key — Sending Your Command

After typing a command, press Enter to run it. Until you press Enter, the computer treats your input as incomplete.

If you make a typo, use Backspace to delete characters. After a command runs, the result appears and a new prompt returns so you can type the next command.

Why Learn Commands?

Terminal commands help in situations like these:

  • Organizing and checking many files
  • Inspecting logs from programs or servers
  • Reading configuration files
  • Working on Linux servers or Mac development environments

From Chapter 1 onward, you will learn commands such as ls (list) and cd (change directory) one step at a time. Take your time — read the explanation before moving to the exercise.
