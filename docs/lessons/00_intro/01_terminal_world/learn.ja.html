English

IZM Linux Dojo へようこそ

ターミナル（端末）という画面を使って、コンピューターへの命令（コマンド）を学びます。文字だけの画面ははじめてかもしれませんが、コマンドがわかると、ファイルの整理や設定の確認など、さまざまな作業を正確に・素早くこなせるようになります。

第0章では、ターミナル画面の見方と、Linux や macOS との関係を説明します。

Linux と macOS とは？

Linux（リナックス）は、世界中で使われているオペレーティングシステム（OS）のひとつです。サーバー、スマートフォン（Android）、組み込み機器など、身の回りのコンピューターの多くが Linux 系の OS で動いています。

macOS（マックオーエス）は、Apple の Mac パソコンで使われている OS です。見た目は Linux とは違いますが、内部の仕組みは Linux とよく似た Unix 系です。だから、ターミナルでの基本操作やファイルの考え方は、Linux と macOS でほとんど同じです。

以降のレッスンでは Linux を主な例にしつつ、Mac をお使いの方も同じ感覚で学べるように説明します。

※ Windows はファイルのパスや権限の仕組みに加え、使えるコマンドも大きく異なるため、本カリキュラムの対象外です。

GUI と CUI — 2 つの操作のしかた

コンピューターを操作する方法は、大きく 2 種類あります。

  • GUI（Graphical User Interface）— グラフィカル・ユーザー・インターフェース。アイコンやボタン、マウスで操作する、いつもおなじみの画面のことです。Finder（Mac）やファイルエクスプローラー（Windows）でフォルダを開くのも GUI です。
  • CUI（Character User Interface）— キャラクター・ユーザー・インターフェース。文字（テキスト）だけの画面で、キーボードから命令を入力する方式です。ターミナルは CUI の代表例です。

GUI は直感的でわかりやすい反面、同じ作業を何度も繰り返すときや、細かい条件を指定するときは手間がかかることがあります。CUI（ターミナル）は最初は慣れが必要ですが、短い命令で正確な操作ができるのが強みです。

ターミナル画面の見方

演習ページでは、画面の左側（または中央）にターミナルが表示されます。典型的なターミナルは次のような構成です。

Welcome to IZM Linux Dojo
Type commands and press Enter.

user@dojo:~$ _
  • Welcome メッセージ — 挨拶の表示です。読んで Enter を押す必要はありません。
  • プロンプト（prompt）— user@dojo:~$ のような、入力待ちを示す表示です。ここから右に、あなたがコマンドを入力します。
  • カーソル — 点滅する _ の位置に、次に入力する文字が入ります。

プロンプトの ~ は、あなたのホーム（自分専用のフォルダ）にいることを表す記号です。詳しくは第1章で学びます。

Enter キー — 命令を送る合図

コマンドを入力したら、Enter キーを押して実行します。Enter を押すまで、コンピューターは「まだ入力の途中」とみなします。

間違えて入力したときは、Backspace（←）で文字を消せます。実行後に結果が表示され、また新しいプロンプトが現れて、次のコマンドを入力できます。

なぜコマンドを学ぶの？

ターミナルで学ぶコマンドは、次のような場面で役立ちます。

  • たくさんのファイルを整理・確認するとき
  • プログラムやサーバーのログ（記録）を調べるとき
  • 設定ファイルの内容を確認するとき
  • Linux サーバーや Mac の開発環境で作業するとき

第1章では、ファイルやフォルダの仕組みを理解したうえで、ls（一覧表示）や cd（移動）を学びます。
