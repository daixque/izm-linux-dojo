「誰の」ファイルか

第1〜3章では、ターミナルでファイルやディレクトリを操作する方法を学びました。ls で一覧を見たり、cat で中身を読んだり、mkdir や touch で新しく作ったりしました。

Linux では、ファイルには「誰のものか」という情報が付いています。同じフォルダに並んでいても、持ち主が違えば触れる操作が変わることがあります。この章では、その「誰」＝ユーザーについて見ていきます。

ユーザーアカウントとは

Linux や macOS では、コンピューターを使う人ごとにユーザーアカウントが割り当てられます。ログインするときの「ユーザー名」が、まさにこのアカウントです。

たとえば、学校の PC を太郎さんと花子さんが順番に使う場合、それぞれ別のユーザー名でログインします。太郎さんが作ったファイルと、花子さんが作ったファイルは、通常は別々に管理されます。

演習では user というユーザー名を使います。ホームディレクトリは /home/user です。macOS でも同じ考え方で、ホームは /Users/あなたの名前 になります。

今ログインしているユーザーを確認する

「今、自分はどのユーザーとして操作しているか」を調べるコマンドが whoami です（"who am I"＝「私は誰？」の略）。

$ whoami
user

ターミナルのプロンプト（入力待ちの行）にも、ユーザー名が表示されています。user@dojo:~$ の user の部分がユーザー名です。

なぜユーザーを意識するのか

ファイルには「持ち主」がいます。次のレッスンで学ぶオーナー（owner＝持ち主）とは、まさにそのユーザーのことです。

  • 自分が作ったファイル → 自分（user）がオーナー
  • 他人が作ったファイル → 他人がオーナー
  • システムが作ったファイル → 管理者の root がオーナーのことが多い

「誰のファイルか」によって、読める・書ける・消せるかが変わります。まずは「ユーザー＝ファイルの持ち主の候補」というイメージを持っておきましょう。
