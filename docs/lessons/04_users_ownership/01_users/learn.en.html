Whose File Is It?

In Chapters 1–3, you learned how to work with files and directories in the terminal — listing with ls, reading with cat, creating with mkdir and touch, and more.

On Linux, files carry information about who they belong to. Even in the same folder, different owners can mean different allowed actions. This chapter looks at that "who" — the user.

What Is a User Account?

On Linux and macOS, each person who uses the computer gets a user account. The username you use to log in is that account.

For example, if two students share a school PC, each logs in with a different username. Files created by one student are usually kept separate from the other's.

In the exercises, the username is user, and the home directory is /home/user. On macOS the idea is the same — home is something like /Users/yourname.

Checking Who You Are Logged In As

The command whoami ("who am I") shows which user you are operating as right now.

$ whoami
user

Your username also appears in the terminal prompt. In the example user@dojo:~$, the user part is the username.

Why Does the User Matter?

Every file has an owner. The owner you will learn about next is exactly that user.

  • Files you create → you (user) are the owner
  • Files others create → they are the owner
  • System files → often owned by the administrator root

Who owns a file affects whether you can read, write, or delete it. For now, remember: user = a candidate owner of a file.
