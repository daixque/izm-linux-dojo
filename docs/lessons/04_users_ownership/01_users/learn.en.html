In Chapters 1–3, you learned how to operate on files and directories in the terminal — listing with ls, reading with cat, creating with mkdir and touch, and more.

From here, we shift perspective. On Linux, it matters whose file something is. In Chapter 4, we learn about that "who" — the user.

What Is a User Account?

On Linux and macOS, each person who uses the computer gets a user account. The username you use to log in is that account.

For example, if two students share a school PC, each logs in with a different username. Files created by one student are usually kept separate from the other's.

In this dojo, your username is user, and your home directory is /home/user (on macOS the idea is the same — home is something like /Users/yourname).

Checking Who You Are Logged In As

The command whoami ("who am I") shows which user you are operating as right now.

$ whoami
user

Your username also appears in the terminal prompt. In this dojo it looks like user@dojo:~$ — the user part is your username.

Why Does the User Matter?

Every file has an owner. The owner you will learn about next is exactly that user.

  • Files you create → you (user) are the owner
  • Files others create → they are the owner
  • System files → often owned by the administrator root

Who owns a file affects whether you can read, write, or delete it. Chapters 5 and 6 cover that in detail. For now, remember: user = the person who owns a file.
