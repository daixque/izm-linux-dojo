Sharing a Computer Among Users

📚 Learn

Review

You can read a file's owner and group from columns 3 and 4 of ls -l. Now imagine a world where many users share one computer.

One PC, Many Users

On a school or lab Linux server, many users share the same machine. Each has a home directory and usually works inside it.

/home/
├── user/       ← you (the dojo exercise user)
├── alice/      ← another user alice
└── bob/        ← another user bob

On macOS, a family Mac works the same way — each person gets something like /Users/name.

Files from Different Owners Together

In shared folders, files with different owners may sit side by side:

$ ls -l shared/
-rw-r--r-- 1 user  user  10 Jan  1 10:00 my_report.txt
-rw-r--r-- 1 alice alice 15 Jan  1 10:00 alice_notes.txt
-rw-r--r-- 1 bob   bob   12 Jan  1 10:00 bob_data.txt

Compare whoami (which prints user) with the owner column in ls -l to tell your files from someone else's.

Others' Files Are Often Off Limits

Files owned by someone else are often unreadable, unwritable, or undeletable. Before touching a file, compare whoami with the owner column from ls -l.
