前のレッスンのおさらい

ls -l の 3 列目・4 列目から、ファイルのオーナーとグループを読めるようになりました。今回は、1 台の PC に複数のユーザーがいる世界を想像してみます。

1 台の PC を複数人で使う

学校や研究室の Linux サーバーでは、たくさんのユーザーが同じコンピューターを共有します。それぞれにホームディレクトリがあり、基本的には自分のホームの中で作業します。

/home/
├── user/       ← あなた（この道場の演習ユーザー）
├── alice/      ← 別のユーザー alice の領域
└── bob/        ← 別のユーザー bob の領域

macOS でも、家族で使う Mac ならユーザーごとに /Users/名前 が分かれているのと同じ考え方です。

他人のファイルが混在する

共有フォルダなどでは、オーナーが違うファイルが同じ場所に並ぶことがあります。

$ ls -l shared/
-rw-r--r-- 1 user  user  10 Jan  1 10:00 my_report.txt
-rw-r--r-- 1 alice alice 15 Jan  1 10:00 alice_notes.txt
-rw-r--r-- 1 bob   bob   12 Jan  1 10:00 bob_data.txt

whoami の結果（user）と ls -l のオーナー列を比べると、「これは自分のファイルか、他人のファイルか」が分かります。

他人のファイルは触れないことが多い

他人がオーナーのファイルは、読めない・書けない・消せない、ということがよくあります（第5章で詳しく）。まずは ls -l でオーナーを見分ける練習をしましょう。

次のステップ

演習ページで shared フォルダの ls -l を読み、誰のファイルか見分けてみましょう！
