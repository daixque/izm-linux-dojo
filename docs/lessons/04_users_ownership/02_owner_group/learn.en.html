Identifying File Owners and Groups

Review

In the previous lesson, you used whoami to confirm you are logged in as user. Files record such a user as their owner.

What Is an Owner?

Every file and directory has one owner — usually the user who created it.

If user runs touch memo.txt, the owner of memo.txt is user.

What Is a Group?

Files also belong to a group. Groups bundle users who share access — for example, all students in a students group might use the same shared folder. For now, think of the group as "another name shown next to the owner."

Detailed Listing with ls -l

ls -l shows detailed information for each file on its own line (-l means a longer, more detailed list).

$ ls -l
-rw-r--r-- 1 user user    8 Jan  1 10:00 memo.txt
drwxr-xr-x 2 user user 4096 Jan  1 10:00 documents

Each column (from the left):

  • Column 1 — type and permissions (- = file, d = directory; permissions are covered in Chapter 5)
  • Column 2 — link count (you can ignore this for now)
  • Column 3 — owner (user in this example)
  • Column 4 — group (user here too)
  • Column 5 — file size in bytes
  • Rest — date, time, and name

In the exercise, focus on the owner (column 3) and group (column 4).
