オーナーとグループ

📚 説明
English

前のレッスンのおさらい

前のレッスンでは、whoami で「今のユーザー」が user であることを確認しました。ファイルには、このようなユーザーがオーナー（持ち主）として記録されています。

オーナー（owner）とは

各ファイル・ディレクトリには、1 人のオーナーがいます。だいたい「そのファイルを作ったユーザー」がオーナーになります。

たとえば user が touch memo.txt すると、memo.txt のオーナーは user です。

グループ（group）とは

ファイルにはオーナーだけでなく、グループも設定されています。グループは「同じチームのメンバー」をまとめる仕組みです。

たとえば学校の PC では「students グループ」に所属する生徒が、同じ共有フォルダを使える、といった使い方があります。初心者のうちは「オーナーと一緒に表示される名前」くらいで大丈夫です。

ls -l で詳しい情報を見る

第1章で ls -l をちらっと紹介しました。今度は本格的に使います。-l は long format（詳細表示）の意味です。

$ ls -l
-rw-r--r-- 1 user user    8 Jan  1 10:00 memo.txt
drwxr-xr-x 2 user user 4096 Jan  1 10:00 documents

各行の意味（左から）：

  • 1列目 — 種類と権限（- はファイル、d はディレクトリ。権限の読み方は第5章で詳しく学びます）
  • 2列目 — リンク数（ここでは気にしなくて OK）
  • 3列目 — オーナー（この例では user）
  • 4列目 — グループ（この例でも user）
  • 5列目 — ファイルサイズ（バイト）
  • 6列目以降 — 日時とファイル名

演習では、3列目のオーナーと4列目のグループに注目してください。
