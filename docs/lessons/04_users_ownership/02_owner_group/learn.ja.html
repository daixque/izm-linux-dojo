ファイルの持ち主と所属

各ファイル・ディレクトリには、2 種類の「誰に関係するか」の情報が記録されています。

  • オーナー（owner）— そのファイルの持ち主（1 人のユーザー）
  • グループ（group）— そのファイルが属するチーム名

だいたい「そのファイルを作ったユーザー」がオーナーになります。たとえば user が touch memo.txt すると、memo.txt のオーナーは user です。

グループとは

グループは「同じチームのメンバー」をまとめる仕組みです。学校の PC では「students グループ」に所属する生徒が、同じ共有フォルダを使える、といった使い方があります。

個人の PC では、オーナーと同じ名前のグループが付いていることが多いです。初心者のうちは「オーナーと一緒に表示される名前」くらいで大丈夫です。

ls -l で詳しい情報を見る

ls -l は、ファイルの詳しい情報を1 行ずつ表示するオプションです（-l は list の long＝詳しい一覧、という意味です）。

$ ls -l
-rw-r--r-- 1 user user    8 Jan  1 10:00 memo.txt
drwxr-xr-x 2 user user 4096 Jan  1 10:00 documents

各行の意味（左から）：

  • 1列目 — 種類と権限（- はファイル、d はディレクトリ。権限の読み方は次の章で詳しく扱います）
  • 2列目 — リンク数（ここでは気にしなくて OK）
  • 3列目 — オーナー（この例では user）
  • 4列目 — グループ（この例でも user）
  • 5列目 — ファイルサイズ（バイト）
  • 6列目以降 — 更新日時とファイル名

演習では、3列目のオーナーと4列目のグループに注目してください。
