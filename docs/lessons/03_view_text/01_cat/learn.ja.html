テキストファイルの中身

第2章までで、ファイルやフォルダの作成・コピー・移動・削除を学びました。ここからは、ファイルの中身をターミナルで読む方法を見ていきます。

Linux では、設定ファイルやログ、メモなど多くのデータがテキストファイルとして保存されています。テキストファイルは、文字だけを並べたファイルです。Finder でメモ帳やテキストエディタで開けるファイルと同じ種類のものです。

中身を画面に表示できれば、設定の確認やログの調査など、さまざまな作業に役立ちます。

cat — ファイル全体を表示する

cat は、ファイルの内容をそのままターミナルに表示するコマンドです。短いテキストファイルをざっと確認するときに使います。

$ cat messages.txt
Hello!
Welcome to Linux Dojo.

表示したいファイル名を、コマンドの後ろに書きます。

cat ファイル名

ファイルがある場所に移動してから実行するか、documents/readme.txt のようにパスを指定して実行できます。

複数ファイルを続けて表示する

ファイル名を複数並べると、順番に内容が表示されます。

$ cat part1.txt part2.txt

2 つのファイルの内容が、1 つに続けて表示されたように見えます。

長いファイルには向かない

cat はファイル全体を一度に表示します。行数がとても多いファイル（ログなど）では、画面からあふれて読みきれなくなります。

長いファイルの場合は、先頭や末尾の一部だけを見る head・tail、行数を数える wc など、別の方法を使います。これらはこの章のあとのレッスンで扱います。
