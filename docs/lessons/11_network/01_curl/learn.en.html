Fetching Data from the Web

When receiving data from websites or services on the Internet, communication usually follows HTTP. HTTP is the set of rules by which web browsers and servers exchange data in a fixed format. When your browser displays a page, it is fetching data over HTTP behind the scenes.

The location of data is expressed as a URL (Uniform Resource Locator). For example, https://dojo.example/api/hello points to /api/hello on the server dojo.example.

What Are API and JSON?

An API (Application Programming Interface) is a window through which programs exchange data. Instead of a web browser, you can query an API from the terminal and receive data.

One format APIs often return is JSON (JavaScript Object Notation) — a human-readable text format using key-value pairs.

{"message":"Hello from Dojo API"}

In the example above, the key message has the value Hello from Dojo API.

curl — Fetch from the Terminal

curl is the standard terminal command for fetching web data. Specify a URL and the server's response appears in the terminal. The dojo does not connect to the real Internet — it returns fixed mock responses.

curl URL — Print the Response

$ curl https://dojo.example/api/hello
{"message":"Hello from Dojo API"}

A greeting in JSON format comes back from the mock API.

curl -o file URL — Save to a File

The -o option saves the response to a file — useful together with file operations from Chapter 2.

$ curl -o info.txt https://dojo.example/data/info.txt
$ cat info.txt
IZM Linux Dojo
Version 1.0
