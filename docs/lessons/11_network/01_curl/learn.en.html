Networks and HTTP

When you fetch data from websites or APIs on the Internet, communication usually follows HTTP. Your browser also uses HTTP behind the scenes to load pages.

curl is a common terminal command for fetching web data. In the dojo, no real network connection is made — fixed mock responses are returned instead.

curl URL — Print the Response

$ curl https://dojo.example/api/hello
{"message":"Hello from Dojo API"}

Specify a URL and the response body is printed in the terminal.

curl -o file URL — Save to a File

The -o option saves the response to a file — useful together with file operations from Chapter 2.

$ curl -o info.txt https://dojo.example/data/info.txt
$ cat info.txt
IZM Linux Dojo
Version 1.0

URLs Available in the Dojo

  • https://dojo.example/api/hello — greeting JSON
  • https://dojo.example/data/info.txt — text info
  • https://dojo.example/api/status — status JSON
