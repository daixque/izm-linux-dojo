Running Programs in the Background

📚 Learn
日本語

Two Ways to Run

When you run a command in the terminal, you usually wait until it finishes. This is running in the foreground — work happening in front of you.

Adding & at the end runs the command in the background. The terminal becomes available immediately while processing continues behind the scenes.

$ sleep 100 &
[1] 3099

[1] is the job number; 3099 is that process's PID.

jobs — List Background Jobs

The jobs command shows work running in the background. In the dojo simulator, you will see fixed output like:

$ jobs
[1]+ Running    sleep 100 &

[1] is the job number; Running means it is active.

fg — Bring to the Foreground

fg (foreground) brings a background job back to the front. Once in the foreground, you wait for that command to finish again. The dojo does not fully simulate this, but it is used in server administration when you want to check a process that was running in the background.

When to Use Background?

  • Run a long task in the background while using the terminal for other work
  • Keep a server program running while typing other commands
