Two Ways to Run

When you run a command in the terminal, you usually wait until it finishes. This is running in the foreground.

Adding & at the end runs the command in the background. The terminal becomes available immediately while work continues behind the scenes.

$ sleep 100 &
[1] 3099

jobs — List Background Jobs

The jobs command shows background work. In the dojo simulator, you will see fixed output like:

$ jobs
[1]+ Running    sleep 100 &

[1] is the job number; Running means it is active.

About fg (Concept)

fg brings a background job back to the foreground. The dojo does not fully simulate this, but remember: it returns background work to the state where you wait for it in front.

When to Use Background?

  • Run a long task in the background while using the terminal for other work
  • Keep a server program running while typing other commands

Chapter 10 is optional. Understanding the idea is enough; try & on a real Linux system when you can.
