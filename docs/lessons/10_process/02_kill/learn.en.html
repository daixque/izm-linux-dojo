Stopping a Process

In the previous lesson you used ps to see running processes and their PIDs. Sometimes you need to stop a process that is no longer needed or keeps running by mistake.

On Linux, the kill command sends a signal to a process with a given PID — essentially telling it to stop.

What is a Signal?

A signal is a message from the OS to a process, such as "exit now" or "pause." For beginners, remember that kill PID means "stop the process with this PID."

Basic kill

$ kill 3099

This tells PID 3099 (the sleep 100 process) to exit. On success, nothing is usually printed.

Run ps again and the stopped process will no longer appear.

Cautions

  • Specifying a non-existent PID produces an error
  • Important processes like systemd (PID 1) cannot be stopped
  • In production, always confirm before stopping a process
