English

プロセスを止める

前のレッスンで ps を使い、動いているプロセスと PID を確認する方法を学びました。不要になったプロセスや、意図せず動き続けているプロセスを止める必要があるときがあります。

Linux では kill コマンドで、指定した PID のプロセスに「終了しなさい」というシグナル（signal）を送ります。

シグナルとは？

シグナルは、OS がプロセスに送る「お知らせ」です。たとえば「終了して」「一時停止して」といった指示を、プログラムに伝えます。

初心者向けには、kill PID が「その PID のプロセスを終了させる」という意味だと覚えれば十分です。

kill の基本

$ kill 3099

PID 3099 のプロセス（sleep 100）に終了を指示します。成功すると、通常は何も表示されません。

もう一度 ps を実行すると、終了したプロセスは一覧から消えます。

注意点

  • 存在しない PID を指定するとエラーになる
  • systemd（PID 1）など、重要なプロセスは止められない
  • 本番環境では、止める前に本当に不要か確認する
