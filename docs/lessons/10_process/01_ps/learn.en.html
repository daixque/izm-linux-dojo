What Is a Process?

A process is a program that is currently running on the computer. When bash runs in a terminal, when you open a web browser, or when a background update runs — each is a separate process.

Think of files (from Chapter 1) as stored data, and processes as work happening right now. Files remain on disk; processes exist only from start until they finish.

Process ID (PID)

The OS assigns each running process a number called a Process ID (PID), such as 1042 or 3099.

  • Starting the same program twice gives two different PIDs
  • When a process exits, its PID may later be reused by another process

In the next lesson, you will specify this PID when stopping a process.

ps — List Processes

The ps command shows which processes are running. In the dojo simulator, fixed sample data is displayed.

$ ps
PID TTY          TIME CMD
  1 ?           00:00:01 systemd
1042 pts/0       00:00:00 bash
2048 pts/0       00:00:02 python simulator.py
3099 pts/0       00:00:00 sleep 100

Main column meanings:

ColumnMeaning
PIDProcess ID (number)
TTYWhich terminal it is attached to (? means none)
TIMEApproximate CPU time used
CMDCommand name

About top (Reference)

On real Linux, top shows a live-updating process list. The dojo cannot fully reproduce that, so we focus on checking the list with ps.
