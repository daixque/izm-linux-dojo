# IZM Linux Dojo カリキュラム

## 教材の方針

**このプロジェクトでは、Python 道場以上に「概念の説明」が学習の中心である。**

学習者は **コンピューター自体の操作に不慣れな初心者** である。ターミナル、ファイル、ディレクトリ、パスといった用語すら初めて触れる人が想定読者なので、当たり前の前提を置いてはならない。

- **説明（learn）が主、演習（exercise）が従** — 理解を深めるのは説明ページの役割であり、演習はその確認・定着のための補助である
- **ベースラインから丁寧に** — 各レッスンは、その時点で必要な概念をゼロから説明する。いきなりコマンドの使い方だけを並べない
- **これまでの学習を積み上げる** — 新しい内容は、前の章・前のレッスンで習ったことを明示的に参照しながら導入する
- **用語・比喩・具体例を重視する** — 抽象語は定義してから使う。図やツリー表示、日常のたとえでイメージを持たせてから操作に入る
- **演習は「できた」の確認** — 説明で身につけるべき理解を、ターミナル操作で試す場として設計する。説明を読まずに演習だけで学べる構成にしない

レッスンを書く・レビューするときは、常に「コンピューター初心者が、この説明だけで迷わず読めるか」を基準にする。執筆時の追加注意（指摘応答文の禁止など）は [`AGENTS.md`](AGENTS.md) を参照。

**第1章以降の learn ページでは**、シミュレータが Tab 補完・コマンド履歴（↑↓）に対応していることを早い段階で触れ、正式なレッスン（第9章）まで待たせない。

---

ターミナルで Linux コマンドを学ぶ Web 学習サイト。中学生〜初心者向けに、ファイル操作からシェルの基本まで段階的に学べる構成とする。

学習者は Linux だけでなく **macOS** など Unix 系 OS を使っている場合もある。ホームディレクトリやターミナルの感覚は共通するので、説明では両方に触れる。**Windows** はファイルパスや権限の体系が異なるため、本カリキュラムの対象外とする。

---

## カリキュラム全体像

難易度は **操作の単純さ** と **抽象概念の深さ** の 2 軸で段階を上げる。類似教材（Progate、Junior Admin、Penguin Gym 等）と同様、**権限はパイプより前** に配置する。

| 区分 | 章 | 焦点 | 主な抽象概念 |
|------|-----|------|--------------|
| **必修** | 第0–3章 | 入口・移動・ファイル操作・テキスト表示 | ターミナル、パス、ファイル |
| **必修** | **第4–6章** | **ユーザー・権限・特権** | **オーナーシップ、rwx、root、sudo** |
| **必修** | 第7–9章 | 検索・パイプ・シェル | grep、リダイレクト、man/help |
| **発展（任意）** | 第10–11章 | プロセス・ネットワーク | PID、HTTP（概念中心） |
| **必修** | 第12章 | 総合演習 | — |

### 学習順序の原則

1. **第0章** — Linux・ターミナルとは何か（いきなりコマンドを打たない）
2. **第1–3章** — 場所の移動とファイル操作（触って覚える土台）
3. **第4–6章（権限ブロック）** — grep やパイプより先に「誰のファイルか」「何が許されているか」を理解する
4. **第7–8章** — 検索・パイプ（権限の理解があると Permission denied も腑に落ちる）
5. **第9章** — シェルの便利機能と **`man` / `--help`**（自分で調べる力）
6. **第10–11章** — 発展編（時間があれば。中学生の必修範囲外）
7. **第12章** — 総合演習

**第4–6章（権限ブロック）の内部順序**

1. **第4章** — 「誰のファイルか」（ユーザー・オーナー・グループ）
2. **第5章** — 「何が許されているか」（rwx と `chmod` 記号形式）
3. **第6章** — Permission denied のあとで **root** / **sudo** / `chown` を導入

`root` や `sudo` を権限の説明より先に出すと、初心者は「なぜ必要か」を理解できない。必ず **オーナーシップ → 権限 → 拒否の体験 → 特権** の順とする。

## ブラウザターミナルの仕組みと対応範囲

演習ページでは **xterm.js + 仮想ファイルシステム（VFS）+ シェルシミュレータ** により、ブラウザ上でコマンドを実行できる。テストボタンで模範解答相当のコマンド列が正しく動くかを自動判定する。

### 現在実装済み

| 機能 | 内容 |
|------|------|
| コマンド | `ls`, `cd`, `pwd`, `cat`, `clear` |
| VFS | 読み取り（一覧・内容表示・パス解決）、YAML で初期ファイルツリーを定義 |
| ターミナル UI | コマンド履歴（↑↓）、Tab 補完（コマンド名・パス）、ファイルエクスプローラー連動 |
| テスト | `command_output`（出力一致）、`cwd`（作業ディレクトリ）、`command_sequence`（手順の逐次検証） |

### カリキュラム実現のための追加実装（予定）

| カテゴリ | 必要な実装 |
|----------|------------|
| ファイル作成・変更 | VFS への書き込み API + `mkdir`, `touch`, `rm`, `cp`, `mv` |
| `ls` オプション | `-a`（隠しファイル）、`-l`（詳細表示・オーナー・権限） |
| ユーザー・オーナーシップ | VFS ノードに `owner`, `group` 属性 + 現在ユーザー概念 + `whoami` |
| 権限 | VFS に `mode` 属性 + 権限チェック + `chmod` |
| 特権操作 | `sudo` シミュレーション（一時的に root 権限）+ `chown`, `chgrp` |
| テキスト操作 | `head`, `tail`, `wc`, `grep` |
| パイプ・リダイレクト | シェルパーサー拡張（`\|`, `>`, `>>`, `<`） |
| 検索 | `find`（VFS 走査）、glob 展開（`*`, `?`） |
| ヘルプ | `man`（固定ページ）または `--help` フラグ |
| プロセス・ネットワーク | **固定レスポンスのシミュレーション**（`ps`, `kill`, `curl` 等。実 OS 連携はしない） |

### ブラウザでは扱わない（カリキュラム対象外）

- 実ネットワーク通信（`ping`, `ssh`, パッケージ管理 `apt` 等）
- エディタ（`vim`, `nano`）の本格操作
- ユーザーアカウントの追加・削除（`useradd`, `usermod` 等）— 概念としてユーザーは学ぶが、管理操作は対象外
- システムサービス（`systemctl`）、マウント等のシステム管理
- スクリプト実行（`bash script.sh`）— 将来の拡張候補
